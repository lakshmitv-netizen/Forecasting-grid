Critical Issues (3)

Issues that block access for users with disabilities. Must fix before release.

1. Grid Cells Not Accessible via Keyboard

Critical WCAG 2.1.1

Grid data cells lack proper keyboard navigation. Users cannot Tab through cells or use arrow keys to navigate the grid.

Affected: HierarchicalGrid, DimensionsTimeGrid, TimeDimensionsGrid

Impact: Screen reader and keyboard-only users cannot access forecast data

// Current: No keyboard navigation <div className="grid-cell" onClick={handleClick}> // Required: Add tabIndex and keyboard handlers <div className="grid-cell" role="gridcell" tabIndex={0} onClick={handleClick} onKeyDown={handleKeyDown} aria-label={`${measure} for ${product} in ${month}: ${value}`} >

2. SVG Icons Missing Accessible Names

Critical WCAG 1.1.1

125 SVG icons found without accessible text. Screen readers announce nothing or "image" for these icons.

Affected: All icon buttons throughout the application

// Current: No accessible name <button><svg>...</svg></button> // Required: Add aria-label or title <button aria-label="Settings"> <svg aria-hidden="true">...</svg> </button>

3. Context Menu Not Keyboard Accessible

Critical WCAG 2.1.1

Right-click context menu cannot be accessed via keyboard. No keyboard shortcut or alternative method provided.

Affected: CellContextMenu.tsx

Fix: Add Shift+F10 or dedicated menu button for keyboard access

Serious Issues (7)

Issues that significantly impact usability. Should fix before release.

# Issue WCAG Description & Fix
4 Missing Focus Indicators 2.4.7 Many interactive elements lack visible focus states. Only 33 :focus styles found for 93 interactive elements. Add outline or box-shadow on :focus.
5 No Skip Link 2.4.1 No "Skip to main content" link. Users must Tab through entire header/nav to reach grid. Add skip link as first focusable element.
6 Missing Landmark Roles 1.3.1 No <main>, <nav>, <aside> landmarks. Side panels should be <aside>, grid area should be <main>.
7 Form Labels Not Associated 1.3.1 59 <label> elements found but not all properly associated with inputs via htmlFor/id. Screen readers may not announce labels.
8 Modal Focus Trap Missing 2.4.3 ReorderMeasuresModal doesn't trap focus. Users can Tab outside modal while it's open. Implement focus trap.
9 No Live Regions 4.1.3 Status updates (e.g., "Last refreshed", cell edit confirmations) not announced. Add aria-live regions for dynamic content.
10 Color-Only Information 1.4.1 Edit history indicators, selection states use color alone. Add icons or text patterns for colorblind users.

Remediation Roadmap

Phase 1: Critical Fixes (1-2 weeks)

  • Add keyboard navigation to grid (arrow keys, Enter to edit)
  • Add aria-label to all icon buttons
  • Implement keyboard access for context menu (Shift+F10)
  • Add role="grid", role="row", role="gridcell" to data grid

Phase 2: Serious Fixes (2-3 weeks)

  • Add visible focus indicators to all interactive elements
  • Implement skip link navigation
  • Add landmark roles (main, nav, aside)
  • Associate all labels with form controls
  • Implement focus trap in modals
  • Add aria-live regions for status updates
  • Add secondary indicators beyond color

Phase 3: Moderate & Minor Fixes (1-2 weeks)

  • Fix color contrast issues
  • Add aria-expanded to expandable elements
  • Add proper ARIA roles to dropdowns
  • Increase touch target sizes to 44x44px minimum
  • Respect prefers-reduced-motion
  • Update page titles on route changes
